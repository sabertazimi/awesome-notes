GraphQL Basic Notes

[TOC]

Better REST#

  • No OverFetching
  • No UnderFetching (need multiple RESTful request)
  • Flexible (clients can specify their exact data requirements)
  • strong Schema and Type System (just like database)

GraphQL Client#

  • ctrl+enter: run query
  • ctrl+space: auto complete in () or {}
  • shift+ctrl+p: prettify

Basic Query Syntax#

{  site {    id    siteMetadata {      author    }    host    port    pathPrefix    buildTime  }}
{  "data": {    "site": {      "id": "Site",      "siteMetadata": {        "author": "sabertazimi"      },      "host": "localhost",      "port": "8000",      "pathPrefix": "/react-blog",      "buildTime": "2019-03-05T13:40:39.129Z"    }  }}
